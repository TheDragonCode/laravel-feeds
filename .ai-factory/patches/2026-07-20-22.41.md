# Coverage gate regression after CSV serialization changes

**Date:** 2026-07-20 22:41
**Files:** tests/Unit/Converters/CsvConverterTest.php, tests/Feature/Feeds/PublicationSafetyTest.php
**Severity:** low

## Problem

The full test suite passed, but `composer test:coverage` failed because aggregate coverage dropped to 94.8% below the required 95.0% threshold.

## Root Cause

The CSV implementation added schema validation and failure-safe generator paths without covering the missing-column branch or a conversion failure while the initial draft was open. The new executable statements lowered aggregate coverage even though functional tests remained green.

## Solution

Added a CSV schema regression test for a row with missing columns. Added an integration test that rejects nested CSV info, verifies the open draft is closed and staging is removed, and asserts the existing DEBUG failure context. The full coverage gate now reports 95.3%.

## Prevention

- Run the exact coverage gate before publishing changes that add executable branches.
- Cover both equal-width schema mismatches and missing or extra columns.
- Exercise failure paths at resource ownership boundaries, including cleanup and diagnostic output.
- Keep coverage improvements behavior-focused and avoid coverage-ignore annotations.
